NAME

Perl::Critic::Policy::Subroutines::ProhibitAmpersandSigils - Don't call functions with a leading ampersand sigil.

AFFILIATION

This Policy is part of the core Perl::Critic distribution.

DESCRIPTION

Since Perl 5, the ampersand sigil is completely optional when invoking subroutines. And it's easily confused with the bitwise 'and' operator. 

  @result = &some_function(); #Not ok
  @result = some_function();  #ok

CONFIGURATION

This Policy is not configurable except for the standard options.
